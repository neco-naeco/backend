## 1. 서비스 정의 및 기획 배경 (What & Why)

| 항목 | 내용 |  |
| --- | --- | --- |
| 서비스명 | 릴레이코딩게임 |  |
| 서비스 개요 | AI 게임 마스터와 함께 코딩 미션을 수행하는 턴제 릴레이 코딩 게임 플랫폼 |  |
| 기획 배경 | 비대면 코딩 학습 시 혼자 하는 공부는 지루하고, 기존의 페어 프로그래밍은 방식이 딱딱하여 몰입도가 낮음. 코딩에 게임의 재미(Gamification)를 결합하여 학습 효율과 팀워크를 극대화하고자 함. |  |
| 핵심 가치 제안 (Value Proposition) | 재미있는 코딩 학습 + 턴제 기반의 협동/경쟁 + AI 게임 마스터의 미션 수행 |  |

---

## 2. 사용자 정의 (Who)

| 항목 | 내용 |
| --- | --- |
| 타겟 사용자 | 코딩을 취미로 즐기고 싶은 개발자, AI 이후 직접 코드를 작성하는 재미를 다시 느끼고 싶은 사람 |
| 사용자 페르소나 |   • 개발 경험이 있는 비전공자 또는 주니어 개발자
  • AI 도구 활용에 익숙해지며 직접 코딩하는 빈도가 줄어듦
  • 과거 직접 구현하던 재미와 몰입감을 다시 느끼고 싶음 |
| 핵심 활동 (Job to be Done) | - AI 게임 마스터와 대화하여 게임룸 생성 및 미션 수락
- 팀원과 턴을 나누어 코드 완성
- 결과 판정 및 리플레이 확인 |

---

## 3. 문제 분석 (Problem Analysis)

| 항목 | 내용 |
| --- | --- |
| 현재 문제 | AI 도구의 확산으로 개발 과정에서 직접 코드를 작성하는 비중이 줄어들고,코딩을 ‘결과를 얻기 위한 수단’으로만 인식하게 되면서 몰입감과 재미가 감소함. |
| 기존 방식의 한계 | - 코드 작성 과정보다 결과 중심으로 흐르며 직접 구현 경험이 줄어듦
- 코드, 실행, 공유가 분리되어 있어 흐름이 끊기고 몰입도가 떨어짐
- 에러 해결을 각자 AI나 검색에 의존하며 협업 기반 학습이 약화됨 |

---

## 4. 해결 전략 (Solution Approach)

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | 채팅창 중심(Chat-first)의 인터페이스 + 턴제 릴레이 엔진 + Docker 기반 미션 환경 |
| 접근 방식 | - Chat-first UI**:** 게임 조작(생성/초대/진행)을 AI 채팅 명령어로 통합.
- WebSocket 기반 실시간 릴레이 엔진
- LLM 기반 에러 분석 및 가이드 제공 |
| 차별화 포인트 | - AI 채팅으로 게임룸 생성, 친구 초대
- 코드 +  에러 로그 동시 공유
- AI 기반 자동 디버깅 및 해결 가이드 제공 |

## 6. 핵심 기능 정의 (Core Features)

| 기능명 | 설명 | 사용자 가치 |
| --- | --- | --- |
| AI 마스터 (Chat-first) | AI 채팅을 통한 게임룸 생성, 초대, 설정 관리 | 별도 UI 없이 대화만으로 게임 운영 |
| 턴제 릴레이 엔진 | 현재 턴 사용자만 편집 가능 (Lock/Unlock) | 릴레이 게임의 긴장감 극대화 |
| 게임 상태 관리 | 턴 시간 제한, 스트라이크(Strike) 부여 | 게임의 공정성과 룰 유지 |
| 실시간 게임 콘솔 | 팀원 모두가 공유하는 실시간 실행 로그 | 동일한 결과를 보며 협업 수행 |
| 코드 스냅샷/리플레이 | 턴 종료 시 코드 상태 저장 및 재생 | 게임 복기 및 학습 효과 |
| Docker 기반 미션 환경 | 템플릿 파일 기반의 독립 실행 환경 | 환경 설정 없이 즉시 미션 수행 |
